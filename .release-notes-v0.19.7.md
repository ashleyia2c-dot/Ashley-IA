# v0.19.7 — First public release

This is the **first release of Ashley that actually works on other PCs** (every prior version had a fatal bug that only let it run on the developer's machine). Across the v0.19.0 → v0.19.7 sequence we fixed **9 critical bugs** and added 4 new features. If you're upgrading from anything earlier, install fresh — the architecture changed significantly.

## ✨ New features

### 7 languages with cultural adaptation
Ashley now speaks **English, Español, Français, 日本語, Deutsch, Русский, 한국어** — not machine translation, but personality and tone adapted per culture (e.g. Ashley calls the user 「ご主人」 in Japanese, "Chef" in German, «шеф» in Russian, 「오빠」 in Korean). Welcome message, achievements, manual, settings, and her own dialogue all respect your language.

### Ashley in 3D (VRM)
- Full 3D character with three-vrm + MToon shader
- **Real lipsync** — mouth syncs to actual TTS audio amplitude, not random
- Eye tracking follows your cursor, blinks naturally
- Head bob while talking, expression boost at end of sentences
- Proximity smile when cursor is near her
- Performance optimized: amp 25fps, cursor 20fps, render 30fps

### Language picker in Settings
7 buttons with native names — click to change instantly.

### Branded installer + uninstaller wizard
Full setup flow with Ashley icon, Welcome → Install Location → Install → Finish. Same for uninstall.

## 🐛 Critical fixes

- **Installer now works on any Windows x64 PC** (was previously broken since project start — the bundled venv had absolute paths to the dev's Python interpreter, replaced with portable Python embeddable distribution)
- `python -m reflex` invocation instead of `reflex.exe` shim (the shim also had absolute paths baked in)
- `cloudflared.exe` properly unpacked from app.asar
- TTS no longer reads roleplay text (`*sonríe*`) — DOM walker that skips italic markdown
- No more empty bubble after `[check_in_goal:...]` actions
- "Thinking..." indicator now translates to your language (was hardcoded Spanish)
- `.lnk.bak` files no longer pollute the desktop when enabling Browser CDP — backups stored in `%APPDATA%\Ashley\data\browser_lnk_backups\`
- Click outside Settings modal closes it (was stuck open)
- Python backend stderr now logged to `main.log` (was being thrown away in packaged builds)

## 🔒 Privacy & legal

- Privacy Policy + Terms of Service accessible from Settings → Legal section
- Belgium jurisdiction, GDPR-compliant
- All data lives in `%APPDATA%\Ashley\data\` — never sent to our servers
- BYOK (Bring Your Own Key) for xAI / OpenRouter / Ollama — your conversations go directly to your chosen provider, we don't see them

## 📦 Installation

1. Download `Ashley-Setup-0.19.7.exe`
2. Windows may show "Unrecognized publisher" warning — this is normal (no code signing yet). Click "More info" → "Run anyway"
3. Wizard: Welcome → Install location → Install → Finish
4. First launch: ~10-15 seconds while Python + AI models initialize. Subsequent launches ~3-5 seconds.

## 🐛 Known issues

- Windows SmartScreen warning on first install (no code signing certificate yet — will add when revenue allows)
- Privacy Policy / Terms of Service translated to EN/ES only — other 5 languages fall back to EN (legal review needed before translating)
- First-launch openWakeWord model download (~5MB) requires internet
