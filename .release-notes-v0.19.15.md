# v0.19.15 — Stable installer (oneClick mode)

## What changed

**Reverted installer to one-click mode** for guaranteed compatibility across all Windows configurations. The wizard mode introduced in v0.19.6 worked on most PCs but silently closed on some setups (likely related to specific antivirus / UAC / group policy configurations). Since this would affect real customers, prioritized reliability over the branded wizard UX.

### Install experience now

- Double-click `Ashley-Setup-0.19.15.exe`
- Windows SmartScreen warning → "More info" → "Run anyway" (no code signing yet)
- Splash + progress bar (~30 seconds)
- Ashley launches automatically when install completes
- License gate appears → paste your Lemon Squeezy key → activate
- Welcome to Ashley

No wizard pages, no choices required during install. Simpler and more reliable.

## All cumulative fixes since v0.19.7 (the public release)

### Core fixes that make the app actually run on other PCs
- Python embeddable bundle replaces venv (venv had absolute paths to dev's machine — broke on every other PC)
- `python -m reflex` invocation instead of `reflex.exe` shim (shim had absolute paths baked in)
- `cloudflared.exe` properly resolves from `app.asar.unpacked/` with fallback to `%APPDATA%\Ashley\bin\`

### License gate (NEW for production)
- Lemon Squeezy license activation flow
- 7-day offline grace period if LS unreachable
- License key input now legible (16px Consolas, left-aligned, 540px wide modal)
- Email link points to real support address: `ashleyia2c@gmail.com`

### UI / UX improvements
- 7-language support (EN / ES / FR / 日本語 / DE / RU / 한국어) for welcome message, achievements, manual, settings
- Language picker in Settings (was hidden — only editable via JSON file)
- Click outside Settings dialog now closes it
- "Thinking..." indicator translates to user's language
- Backup/export feature in Settings (RGPD Art 20 — right to data portability)
- Privacy Policy + Terms of Service accessible from Settings → Legal section

### Background fixes
- TTS no longer reads italicized roleplay text (`*sonríe*`)
- No more empty bubble after `[check_in_goal:...]` actions
- `.lnk.bak` files no longer pollute desktop when enabling Browser CDP
- Installer no longer kills your VS Code Python interpreter
- Python stderr now logged to `main.log` (was being thrown away in packaged builds)
