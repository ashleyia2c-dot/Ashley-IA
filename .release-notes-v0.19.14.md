# v0.19.14 — Production-ready release

**This is the first release ready for actual sale.** v0.19.7 was the first version that ran on other PCs at all; v0.19.14 closes the remaining gaps for a real commercial launch: license gate, polished installer, legible UI, complete i18n, GDPR compliance.

If you're a customer who just bought Ashley: download `Ashley-Setup-0.19.14.exe` below, run it, paste your Lemon Squeezy license key when prompted, and you're in.

## ✨ New features since v0.19.7

### Lemon Squeezy license gate (NEW — required to use the app)
- License activation flow with offline grace period (7 days if LS unreachable)
- Activation tied to instance ID — supports up to N PCs per license (configurable on LS dashboard)
- Deactivation flow if you want to migrate to another PC

### Backup / Export your data (RGPD Art 20)
- Settings → "Legal & Data" → "Export all my data (.zip)"
- Downloads everything Ashley knows about you (chat, facts, diary, achievements, preferences) as a JSON archive with manifest
- Excludes API keys (DPAPI-encrypted, can't migrate) and license info (PC-bound)

### Onboarding wizard in 7 languages
- EN / ES / FR / 日本語 / DE / RU / 한국어 — each with culturally adapted tone matching Ashley's prompts
- Welcome message, achievements, manual, settings, license_gate links — all 7 languages

## 🐛 Critical fixes since v0.19.7

- **`cloudflared.exe ENOENT`** at first launch — path-rewrite fix routes spawn to `app.asar.unpacked/` correctly + fallback download to `%APPDATA%\Ashley\bin\` if needed
- **Installer wizard closing silently** when an orphaned install dir from a prior failed install was present — new `CleanupOrphanInstallDir` macro detects + removes it before installing
- **Installer no longer kills your IDE's Python/Node processes** — old taskkill was unfiltered (would kill VS Code's Python interpreter, IntelliJ Node servers, etc); now PowerShell filter only kills processes whose CommandLine contains `ashley\resources`
- **License key input was illegible** — was 11px font with `text_align="center"` so 36-char Lemon Squeezy keys appeared as garbled middle-clipped text. Now 14px Consolas, left-aligned, in a wider 540px modal — keys display fully legibly
- **Support email link was wrong** (`hello@ashley-ia.com` → `ashleyia2c@gmail.com`) — customers losing keys can now actually reach support
- **CI builds were failing** with NSIS warning treated as error — `$_` PowerShell var collided with NSIS `$VAR` syntax; escaped with `$$_`

## 🔒 Privacy & legal

- Privacy Policy + Terms of Service accessible from Settings → Legal & Data (modals with markdown)
- Belgium jurisdiction (Article 8 GDPR — children under 16 not allowed)
- All conversation data lives in `%APPDATA%\Ashley\data\` — never sent to our servers
- BYOK (Bring Your Own Key) for xAI / OpenRouter / Ollama
- Lemon Squeezy is the Merchant of Record for purchases (their privacy policy applies for billing data)

## 📦 Installation

1. Download `Ashley-Setup-0.19.14.exe`
2. Windows SmartScreen will likely show "Unrecognized publisher" — click **"More info"** → **"Run anyway"** (we don't have a code signing certificate yet — coming after first revenue)
3. Wizard: Welcome → Just for me → Install Location → Install → Finish (~30 sec)
4. **First launch shows the license gate** — paste your Lemon Squeezy license key (received by email after purchase) → Activate
5. **Then onboarding** — choose your AI provider (xAI / OpenRouter / Ollama), paste your provider API key (or skip for Ollama)
6. **Chat with Ashley** — she'll greet you in your selected language
